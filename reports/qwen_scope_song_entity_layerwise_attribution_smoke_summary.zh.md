# Qwen 歌曲关系逐层归因 Smoke 报告

## 大白话结论

这次终于看到了一条比较清楚的内部过程：

> 模型不是一看到歌名就立刻得到歌手答案。前半段主要保留和搬运信息，中后段开始把
> 歌名知识汇总到即将输出歌手的位置，到了第 24 层以后，正确歌手的优势才变得稳定
> 可读。

模型共有 28 层。最终正常概率在 12 个关系中答对 10 个，准确率 `0.833`，通过预注册
的 `0.80` 行为门。所有技术自检也通过，因此可以解释逐层结果。

## 我们具体做了什么

以 `Shape of You` 为例，同时测试：

- 正确歌手 `Ed Sheeran`；
- 错误歌手 `Queen`；
- 真实歌名条件；
- `Untitled Song` 中性歌名条件。

第一种观察把每一层的内部状态直接接到模型自己的输出词表上，看该层是否已经更
支持正确歌手。这里没有训练分类器，也没有选择 SAE 特征。

第二种观察在每一层把真实歌名形成的“准备输出歌手”状态替换成中性歌名状态，然后
让模型继续往后运行。如果后续层还能重新读取真实歌名，正确歌手优势会恢复；如果
不能恢复，说明关系信息已经集中到了被替换的位置。

## 仪器自检

| 检查 | 结果 | 门槛 | 状态 |
|---|---:|---:|---|
| 最终层 target-logit 与真实模型 logit 最大误差 | 0.0078 | 0.02 | 通过 |
| 最后一层替换与中性基线最大误差 | 0.0000 | 0.02 | 通过 |
| 最终 sequence-PMI accuracy | 0.833 (10/12) | 0.80 | 通过 |

这两个误差检查说明逐层读出和状态替换落在了正确的张量位置，不是代码把不同状态
混在一起。

## 关系知识在哪一层变得可读

| 深度 | 正确方向比例 | 平均关系 margin |
|---|---:|---:|
| Layer 1 | 0.583 | 0.070 |
| Layer 9 | 0.667 | 0.254 |
| Layer 15 | 0.667 | 0.188 |
| Layer 20 | 0.583 | 0.250 |
| Layer 23 | 0.667 | 1.228 |
| Layer 24 | 0.750 | 1.617 |
| Layer 25 | 0.833 | 1.956 |
| Layer 26 | 0.750 | 1.824 |
| Layer 27 | 0.750 | 1.577 |
| Layer 28 | 0.833 | 1.839 |

按预注册标准，首次连续三层达到至少 `0.75` 的位置是 **Layer 24**。在此之前，准确
方向大多在 `0.50-0.67` 间波动；Layer 23 以后平均 margin 明显增大。

英文关系在 Layer 25 达到 `1.00`，中文为 `0.667`；最终 Layer 28 中英文都为
`0.833`。这提示中文关系在这个小模型里收敛得更晚，但样本只有各 6 条，只作机制
线索，不作语言能力结论。

## 状态替换告诉了我们什么

`remaining fraction` 表示替换后还剩多少真实歌名带来的首 token 优势。`1.0` 代表
几乎没影响，`0` 代表完全变成中性歌名状态。

| 替换位置 | 剩余比例 |
|---|---:|
| Layer 1 | 1.001 |
| Layer 10 | 0.974 |
| Layer 14 | 0.897 |
| Layer 15 | 0.784 |
| Layer 16 | 0.621 |
| Layer 17 | 0.504 |
| Layer 21 | 0.420 |
| Layer 24 | 0.310 |
| Layer 26 | 0.212 |
| Layer 27 | 0.010 |
| Layer 28 | 0.000 |

可以把它理解成：

1. **Layer 1-13：**替换当前位置影响很小。后面的层仍能回头读取歌名，再把信息恢复
   出来。
2. **Layer 14-17：**关系信息开始明显汇总到输出位置。替换后的损失快速增大。
3. **Layer 18-26：**关系优势逐步固化，后续层能恢复的部分越来越少。
4. **Layer 27-28：**输出决定基本完成。替换后几乎完全退回中性歌名结果。

两条首 token 的真实/中性差异小于 `0.05`，所以标准化剩余比例主要由另外 10 条
关系贡献；原始 patch margin 和最后层一致性仍覆盖全部 12 条。

## 为什么这比 SAE 结果更有用

上一轮 layer-24 SAE 的跨歌曲准确率只有 `0.583`，但这次模型自己的 target-logit
读出在 Layer 24 已达到 `0.75`，Layer 25 达到 `0.833`。

这更支持下面的解释：

> 关系知识确实进入了后层残差流，但它不一定被压缩成少数可跨歌曲复用的 SAE 稀疏
> 特征。模型的输出头能够读取这种分布式状态，之前的 SAE 分类器却读不稳。

所以 SAE 失败不等于模型内部没有关系知识。

## 和“模型是否知道自己在虚构”有什么关系

本轮证明的是：面对真实歌曲事实题，歌名会在中后层对歌手预测产生可测的因果影响。
它仍然没有直接证明模型在自由生成错误配对时知道自己正在虚构。

现在我们已经有了一个更可靠的比较基线：真实关系通常应表现出 Layer 14-17 开始
整合、Layer 24 后稳定可读的轨迹。后续可以观察自由生成的曲库冲突配对是否缺少这
条轨迹，或者是否在明知关系不匹配时仍被生成目标压过。

## 下一步决定

下一步不回到大样本 benchmark，也不立即做 SAE steering。最小且最有信息量的实验
是对 Layer 14、16、18、21、24、27 做组件级替换：

- 单独替换 self-attention 输出，测试歌名信息是不是通过注意力被取回；
- 单独替换 MLP 输出，测试关系知识是不是在 MLP 中被转换或放大；
- 保持同一批 matched controls 和相同首 token margin，不训练新分类器。

这会把“关系知识在后半段形成”进一步缩小为“是 attention 在搬运，还是 MLP 在
加工”。组件定位通过后，再把同样方法用于自由生成的正确配对和曲库冲突配对。
